# Non-native Exception Approval Template

Use when an elevated recommendation retains a non-native, nonstandard, wrapper-based, manual, or surprising implementation path.

```markdown
## Exception Approval

- Reason for deviation:
- Constraint source:
- Why the native/baseline path is unavailable or riskier:
- Compensating controls:
- Rollback proof:
- Monitoring/alerting compensation:
- Operational handoff:
- Owner:
- Risk acceptor:
- Review date:
- Expiry date:
- Plan to return to native/baseline path:
```

Missing exception approval blocks any recommendation to submit the non-native path to Final Gate.
